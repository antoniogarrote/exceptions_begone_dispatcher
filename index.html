Exceptions Begone Dispatcher

What?

A system for dispatching exceptions in 'real time' from a production Rails application to a web browser.
Exceptions Begone original server is open source and was developed at XING AG. The Exceptions Begone Dispatcher is still under development.

How?

  • A web client opens a web socket to the Exceptions Begone Dispatcher
  • A set of Erlang processes in the Exceptions Begone Dispatcher start listening for messages in some queues inside a RabbitMQ queue system.
  • The Exceptions Begone Rails plugin integrates with your application and send any raised exception to the Exceptions Begone Dispatcher using a JSON representation of the exception
  • Some Erlang process in the Exceptions Begone Dispatcher routes the exception to a AMQP exchange in a RabbitMQ queue system
  • RabbitMQ routes the exception to the queues of the listener Erlang processes
  • The listener Erlang processes receive the exception from RabbitMQ and send it back to the web browser through the open web socket
  • The web browser shows the information about the received exception
